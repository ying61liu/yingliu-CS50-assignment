sort1 uses: Bubble sort
How do you know?: When data is ramdom and the data size is large, it is the slowest one. When data is in reversed order, it also takes the longest time to finish running.
When it is in sorted order, it is slightly faster than sort3, selection sort, because it compares two data at a time while sort3 iterates through
each one, thus takes more time.

sort3 uses: Selection sort
How do you know?: It is faster than sort1, bubble sort, when the data is sorted but in reversed order, becasue this algorithm does not compare two adjacent data and switch the order if needed,
instead, it iterates through the data and finds the smallest item in the array, thus it is faster than bubble sort. However, when the data is sorted in ascending order and tha data
size is big, its speed is signficantly slower than sort1, because even if the data is sorted, it still interates through each untouched data as unsorted until it
finds the smallest number.
